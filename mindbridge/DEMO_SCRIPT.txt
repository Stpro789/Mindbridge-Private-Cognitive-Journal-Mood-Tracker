[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Let's look at how the workflow is structured.
First, when a user enters a journal, it passes through our Security Checkpoint node, which scrubs email addresses or phone numbers locally and intercepts any prompt injection attacks.
If clean, the text is routed to the Orchestrator node, which coordinates the analysis.
The orchestrator delegates to two specialized sub-agents: the Reflection Agent, which uncovers key cognitive themes, and the Mood Tracker Agent, which rates the user's emotional tone from 1 to 10.
Both agents can retrieve historical logs from our local MCP Server.
Once analyzed, the workflow moves to the HITL Check Node, where a human-in-the-loop validation is triggered if the mood is extreme.
Finally, the approved draft is saved to local storage via the Final Output Node.
